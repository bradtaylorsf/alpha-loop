## Patterns
- When using Commander.js negatable options (`--no-X`), always check `options.X === false` rather than `options.noX` — Commander sets the positive name to `false`, not a `noX` property
- For complex interactive CLI commands, mock `@inquirer/prompts` at the module level in tests to control the full interactive flow without TTY

## Anti-Patterns
- Don't assume CLI framework option naming from the flag string — verify how the framework actually maps negatable flags to the options object
- Don't write contradictory instructions in AI prompts (e.g., "no fences" + fenced example) even if downstream parsing handles both cases
