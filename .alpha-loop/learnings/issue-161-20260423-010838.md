## Patterns
- Detect local vs frontier by `endpoint_type` on the telemetry record rather than parsing URLs at aggregation time — keeps cost math centralized
- `all-frontier` baseline picks highest-cost cell per stage; makes delta columns meaningful even when some stages have no frontier data (delta=null)
- Token estimation fallback (`output.length/4`, `tokens_in = tokens_out * 1.3`) when agent lacks usage data — acceptable for like-for-like A/B since both arms use same heuristic

## Anti-Patterns
- Don't probe remote endpoints in preflight — restrict `/v1/models` checks to loopback (`localhost`, `127.0.0.1`, `::1`, `*.local`) to avoid leaking or rate-limiting
- Don't rely on session-level logs for routing analysis — per-stage granularity is required to attribute tokens/cost to specific (stage, model) cells
