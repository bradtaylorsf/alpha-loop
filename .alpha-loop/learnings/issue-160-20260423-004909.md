## Patterns
- Per-stage routing config (`routing.stages.*` + `routing.endpoints.*` + `routing.fallback`) cleanly layered on top of legacy `agent:`/`model:` without breaking backwards compatibility
- Runtime state files (escalation-state.json) belong in `.gitignore` alongside existing `.alpha-loop/sessions/` and `.alpha-loop/traces/` entries
- Classify tool errors on a single output via `classifyToolErrors(...).length >= 2` for simple per-turn escalation; reserve `newTurnState`/`shouldEscalate` turn-state API for future streaming use

## Anti-Patterns
- Exporting turn-state APIs (`newTurnState`, `shouldEscalate`) from `escalation.ts` that aren't consumed by `pipeline.ts` — creates dead-code surface that only tests exercise
- Defining `RoutingFallbackMode` variants (`retry`) in config types without implementing them in `spawnStageAgent` — config surface should match implementation
- Adding new routing stages (`summary`) without wiring every call site (post-PR `assumptions` step at `pipeline.ts:1081` still uses raw `spawnAgent`)
