## Patterns
- **Subcommand grouping with `isDefault`**: Using Commander's `{ isDefault: true }` on the `run` subcommand so `alpha-loop eval` works as shorthand for `alpha-loop eval run` — good CLI UX pattern
- **Append-only JSONL for score history**: Simple, git-trackable, no migration needed — effective for time-series data that only grows
- **Filter-then-execute pattern**: Loading all eval cases then filtering by tags/suite/type/step before execution keeps the runner logic clean and composable

## Anti-Patterns
- **Hardcoding service URLs**: HTTP check hardcodes `localhost:3000` — should be configurable per-check in `checks.yaml` for repos using different ports
- **Stub implementations behind full CLI surface**: `evalSearchCommand` lists variants but doesn't execute runs; `evolve` stops after first proposal. Shipping CLI commands that appear complete but aren't can confuse users — better to mark as `(preview)` in help text
