## Patterns
- **Scaffold-then-defer for epics**: When implementing an epic that spans multiple milestones, wire up all CLI commands and module boundaries in M1 but leave later-milestone bodies as documented placeholders. This lets the full command surface exist for testing while keeping scope tight.
- **Trace-first architecture**: Building the trace filesystem storage layer before the eval runner meant eval cases could immediately reference real execution data rather than synthetic fixtures.
- **Config loaded but unused is acceptable scaffolding**: Several commands load config without using it yet — this is intentional forward-wiring that avoids a second pass to thread config through later.

## Anti-Patterns
- **Dead fields in shared types**: Adding `evalScore` to `PipelineResult` before any code sets it creates a field that's always undefined. Better to add type fields when the producer is implemented, not just the consumer.
- **Unescaped globs in minimatch**: Using raw file paths (containing dots) as minimatch patterns can cause false positives. Eval case file patterns should be validated or escaped on capture.
