## Patterns
- Stub-with-TODO for golden patches (`# TODO: backfill — gh pr diff N > golden.patch`) lets the case set ship before all artifacts are backfilled, gated by `isStubPatch()` so scoring degrades gracefully.
- Declarative `scorers:` block in `checks.yaml` per case keeps per-case policy (hard vs informational) co-located with the case rather than in runner code.
- Matrix dry-run path (validate profiles + load cases without executing pipelines) gives fast CI smoke coverage of the wiring without paying execution cost.

## Anti-Patterns
- `runMatrix()` hardcodes `diffSimilarity = null` and `toolErrorRate = 0` (src/lib/eval-matrix.ts:217, 300-305) — helpers exist and are tested but never invoked on real diff pairs/telemetry. Future placeholders should at minimum thread real data even if scoring is informational.
- `parseChecks()` reads `obj.checks` (array) and silently ignores the `scorers:` block — declarative config that the runner doesn't consume creates false confidence; either consume it or don't ship it.
- Shipping `golden.patch` files containing only TODO comments risks the backfill never happening; track backfill as a follow-up issue with the rate-limit reset deadline.
