## Patterns
- **Deprecate-don't-remove**: when superseding a command, add a `log.warn()` deprecation notice and update all doc references while keeping the old command functional. This is a clean, low-risk pattern for CLI evolution.
- **Doc-sweep pattern**: when deprecating or adding commands, update all three doc surfaces (CLAUDE.md, README.md, and init flow suggestions) in one commit to avoid drift.
