## What Failed

## Patterns
- Hardware-detection-gated init prompts (Apple Silicon + RAM threshold) are testable with mocked `os` calls — keeps the prompt opt-in without forking the init flow
- Co-locating ready-to-copy YAML profiles in a single `routing-profiles.md` reference doc is friendlier than scattering them across separate files

## Anti-Patterns
- Avoid inline `node -e` with embedded regex/heredoc for verification — escape rules across shell + JS make it brittle; write a one-shot script file or use `pnpm test` against a real test instead
- Don't introduce new model-ID strings in docs without grepping existing profile/config files first; doc/code drift in copy-paste examples breaks the "copy and run" promise

## Suggested Skill Updates
- `.alpha-loop/templates/skills/docs-sync` (or equivalent): when adding doc examples that reference model IDs, config keys, or profile names, grep the repo for existing canonical values and reuse them verbatim — flag any new identifier introduced only in docs
- Reviewer agent: add a check that any YAML block in `docs/**` whose keys overlap with files in `.alpha-loop/evals/profiles/` uses matching values, or explicitly notes the divergence
