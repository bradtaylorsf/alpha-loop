## Patterns
- **Gated evaluation pipeline**: Step-level evals gate expensive e2e evals for fast discard of bad proposals (~$0.60/2min vs ~$4.60/2hr) — reuse this pattern for any multi-tier validation
- **Filesystem as shared memory**: Storing all iteration results (including discards) to disk so the proposer agent can learn from failures mirrors the autoresearch pattern effectively
- **Surface-level risk control**: Limiting what an autonomous agent can modify (prompts only → prompts+code) is a practical safety gradient for overnight unattended runs

## Anti-Patterns
- **Spec flag names vs implementation**: When the spec says `--iterations` but the implementation uses `--max-iterations`, document the deviation early rather than catching it in review
- **Docs lagging feature scope**: When bonus functionality is added beyond spec, docs should be updated in the same pass rather than requiring a review fix
