# `alpha-loop add` — Quick Issue Creation

## Overview

A new CLI command that lets users quickly create a single GitHub issue from a free-form description. The AI generates a fully structured issue (title, body, labels, milestone) which the user reviews before creation.

## CLI Signature

```
alpha-loop add [options]
  --seed <file>        Read description from file instead of prompting
  --milestone <name>   Override milestone assignment
  --dry-run            Preview without creating GitHub resources
  -y, --yes            Skip confirmation prompts
```

## Flow

### 1. Input

- **Interactive**: Prompt user with `input()` from `@inquirer/prompts` for a free-form description
- **Non-interactive**: Read from `--seed <file>` (requires `--yes` to skip confirmations)
- TTY check: if not interactive and no `--seed`/`--yes`, exit with guidance message

### 2. Context Gathering

Load in parallel:
- Existing milestones via `listMilestones(repo)` — provides title, description, open/closed issue counts
- Project context from `.alpha-loop/context.md` if it exists (generated by `alpha-loop scan`)
- Existing labels via `gh label list` to inform the AI of what's available

### 3. AI Generation

Send a one-shot prompt to the configured agent containing:
- User's free-form description
- Existing milestones (title + description + open issue count)
- Project context summary
- Instruction to return a JSON object:

```typescript
{
  title: string;
  body: string;           // Markdown, well-structured
  labels: string[];       // e.g., ["feature", "p1", "medium"]
  milestone: {
    title: string;        // Existing milestone title, or new one
    description: string;  // Only required if isNew
    isNew: boolean;
  };
}
```

Use `extractJsonFromResponse<T>()` (from `src/lib/planning.ts`) to parse the AI response.

Label conventions the AI should follow:
- **Type**: `bug`, `feature`, `chore`, `docs`, `refactor`
- **Priority**: `p0` (critical), `p1` (high), `p2` (medium), `p3` (low)
- **Complexity**: `trivial`, `small`, `medium`, `large`

### 4. Review

Display a formatted summary:
```
Title:      Fix auth token refresh on expired sessions
Labels:     bug, p1, medium
Milestone:  v2.0 - Auth System (existing)

Body:
  [truncated preview or full body]
```

Then prompt:
1. **Milestone confirmation**: "Milestone: 'v2.0 - Auth System' (existing). Accept? (Y/n/pick)"
   - If user says no: show `checkbox` of existing milestones + "Create new" option
   - `--milestone` flag skips this entirely
2. **Overall confirmation**: `confirm()` to create, or offer `editor()` to edit the body
3. **`--yes` mode**: skip all prompts, create directly

### 5. Creation

In order:
1. If milestone is new: `createMilestone(repo, title, description)` — get milestone number
2. `createIssue(repo, title, body, labels)` — get issue number
3. `setIssueMilestone(repo, issueNumber, milestoneTitle)` — assign milestone
4. If `config.project` is set: `addIssueToProject(owner, projectNum, repo, issueNumber)`
5. Log the created issue URL: `https://github.com/{repo}/issues/{num}`

### 6. Dry Run

With `--dry-run`: display the formatted summary and what would be created, then exit without calling GitHub APIs. Same pattern as `plan --dry-run`.

## File Structure

- `src/commands/add.ts` — Command handler (~150-200 lines)
- Registration in `src/cli.ts` — new `.command('add')` block
- Reuses: `src/lib/github.ts`, `src/lib/config.ts`, `src/lib/context.ts`, `src/lib/planning.ts` (for JSON extraction)

No new library files needed. The AI prompt is built inline in `add.ts` (same pattern as `scan.ts` and `triage.ts`).

## Agent Prompt

The prompt sent to the AI agent:

```
You are an issue writer for a software project. Given a user's description and project context, generate a single well-structured GitHub issue.

## Project Context
{context.md contents or "No project context available."}

## Existing Milestones
{formatted list of milestones with title, description, open issues}

## User Description
{user's free-form input}

## Instructions
- Write a clear, actionable issue title (imperative mood, under 80 chars)
- Write a detailed body in Markdown with: Problem/Goal, Proposed Solution, Acceptance Criteria
- Assign appropriate labels for type (bug/feature/chore/docs/refactor), priority (p0-p3), and complexity (trivial/small/medium/large)
- Recommend the best-fit existing milestone, or propose a new one if none fit
- Return ONLY a JSON object (no markdown fences):

{
  "title": "...",
  "body": "...",
  "labels": ["type", "priority", "complexity"],
  "milestone": {
    "title": "...",
    "description": "...",  // only if isNew
    "isNew": false
  }
}
```

## Error Handling

- AI returns unparseable JSON: log error, show raw response, ask user to try again or file manually
- Milestone creation fails: warn, continue creating issue without milestone
- Issue creation fails: show GitHub error, exit non-zero
- No milestones exist: AI must propose a new one (prompt makes this clear)

## Edge Cases

- Empty description: reject with "Please provide a description"
- Very long description (>5000 chars from `--seed`): pass through, let AI summarize
- Milestone title from `--milestone` doesn't match existing: create as new with AI-generated description
- Labels don't exist in repo: `gh issue create --label` auto-creates them
